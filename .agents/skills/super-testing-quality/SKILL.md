## TDD Pipeline

```
1. RED    →  Write failing test (tdd-workflows-tdd-red)
2. GREEN  →  Write minimal code to pass (tdd-workflows-tdd-green)
3. REFACTOR → Improve without breaking (tdd-workflows-tdd-refactor)
4. REPEAT
```
